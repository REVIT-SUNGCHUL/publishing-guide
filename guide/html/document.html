네이밍 규칙

아이디(id) 네이밍 규칙

종류 구분 잘못된 예 올바른 예
ID Selector 카멜 케이스(camelCase) error_Mesage errorMesage
  • 시작은 영문 소문자, 두 번째 단어부터 첫 번째 문자를 대문자로 처리하는 카멜케이스 기법을 사용한다.
  • 동일한 이름의 id는 문서에서 한번만 사용이 가능하다.
  • 지정된 레이아웃 및 폼객체를 제외하고 id를 사용하지 않는다.
  • 앵커로서 사용되는 엘리먼트는 id 지정이 가능하다.
  • User Interface 동작을 위하여 DOM 선택자로서의 id 지정은 가능하다.
  • 예약어가 있는 경우 예약어를 사용한다.

클래스(class) 네이밍 규칙

잘못된 예 올바른 예
error-Mesage error-mesage
error_Mesage error-mesage
  • class는 케밥케이스(kebab case) 기법을 사용한다.
  • 단어 조합 시 형태+의미+(숫자)+상태 형식으로 작성하며, 영문 소문자, 숫자, -(하이픈)만 사용한다.
  • 시작의 이름은 영문 소문자를 사용하며 두 번째 단어부터 -(하이픈)으로 연결하며 대문자는 사용하지 않는다.
  • -(하이픈)은 2개 이상의 단어를 조합할 때만 사용한다.
  • -(하이픈)을 이용하여 3단계를 초과하여 사용하는 방법은 지양한다.
  • 숫자 없으면 '1'이라는 숫자가 생략된 것으로 간주한다.
  • 다중의 클래시 지정을 허용하며 최대 3개 이상 사용하지 않도록 주의한다. 단, 여백, 넓이, 컬러값을 지정하는 클래스는 예외로 한다.

HTML 규칙

표준 문법 사용

  • HTML은 해당 DTD의 명세에 맞게 작성하며, W3C validation을 통과해야 한다.
  • DTD를 제외한 모든 엘리먼트와 속성은 소문자로 작성한다.
  • 속성의 값은 큰따움표("")로 묶는다.
  • 특수기호는 Entity name을 사용하여 entity 코드로 변환한다. Entity 코드는 ISO-8859-1을 기준으로 한다.
  • 마크업은 w3c html5문법에 준하여 사용한다.
  • 종료 태그는 self-closing tags(br, img, input, meta, link, area, col, embed, source, track, wbr)를 제외하고는 생략하지 않는다.

문서형 선언(Document Type Definition)

문서형을 선언하지 않으면 브라우저 호환성을 확보하지 못함. 문서형을 선언하는 것은 상호 운용성을 보장함.

언어/문자셋(charset) 지정

웹페이지에서 언어(lang 속성)와 문자셋(charset 메타 태그)을 올바르게 설정하는 것은 브라우저가 콘텐츠를 정확하고 올바르게 해석 및 표시하는 데 중요한 역할을 한다.

구분 예제 설명
lang 속성 <html lang="ko">
  • 검색 엔진 최적화 (SEO)

    구글, 네이버 같은 검색 엔진이 페이지 언어를 올바르게 인식하여 검색 결과에 적절히 반영함.

  • 스크린 리더 지원 (접근성)

    화면 낭독기가 언어를 정확하게 인식하여 올바른 발음과 억양으로 읽음.

  • 브라우저 번역 최적화

    브라우저의 자동 번역 기능이 페이지의 언어를 정확하게 감지하여 번역할 수 있음.
charset 메타 태그 <meta charset="UTF-8">
  • 문자 깨짐 방지

    UTF-8을 사용하지 않으면 한글이 깨질 수 있음

  • 빠른 렌더링

    <meta charset="UTF-8">을 <head> 태그의 가장 첫 부분에 배치하면, 브라우저가 문서를 해석하는 속도가 빨라짐.

  • 국제 표준 지원

    UTF-8은 전 세계 대부분의 문자(한글, 영어, 이모지, 특수 문자 등)를 포함하는 표준 문자셋이며, 다국어 지원이 원활함.

Html SEO 최적화

SEO(Search Engine Optimization, 검색 엔진 최적화)는 웹페이지가 구글, 네이버 등 검색 엔진에서 높은 순위에 노출되도록 최적화하는 기법

구분 설명
title 태그(페이지 제목)
  • 검색 결과의 제목에 표시됨
  • 각 페이지마다 고유한 제목을 설정해야 함
  • 핵심 키워드를 포함(너무 길면 잘림, 50~60자 권장)
meta description 태그(페이지 설명)
  • 검색 결과에서 제목 아래 설명 부분
  • 페이지 내용을 간략히 요약 (120~160자 권장)
  • 핵심 키워드를 자연스럽게 포함
  • 사용자를 클릭하도록 유도하는 문구 작성 ("자세한 내용은 여기를 클릭하세요!")
헤딩 태그 (h1~h6)
  • h1은 페이지의 가장 중요한 제목

    h1은 한 페이지에 하나만 사용

  • h2~h6는 본문 내 소제목

    h2~h6을 순차적으로 사용 (h2 → h3 순서로)

  • 제목에 키워드를 포함하면 SEO 점수 향상

    제목 태그만으로도 문서 개요가 이해될 수 있도록 작성
이미지 최적화 (alt 태그)
  • 이미지가 로딩되지 않을 경우 대체 텍스트로 표시
  • 검색 엔진이 이미지 내용을 파악하여 이미지 검색 결과 노출 증가
  • 웹 접근성 향상 (스크린리더 사용자를 위해 필수)
빠른 로딩 속도
  • lazy loading, CSS/JS 압축, 브라우저 캐싱
모바일 최적화
  • 반응형 디자인, meta viewport 설정

파일 인코딩

.html, .css, .js의 파일의 저장 방식을 UTF-8로 설정하며, 프로젝트에 따라 변경한다.

문자셋(charset) Encoding
charset="UTF-8" UTF-8
charset="euc-kr" ANSI or euc-kr(외국에서 지원이 안될 가능성이 있음.)

들여쓰기

  • 코드의 가독성을 높이고 html의 구조를 쉽게 파악하기 위해 들여쓰기 규칙을 통일한다.
  • 1탭 들여쓰기 하며, 1탭의 크기는 공백 4칸으로 설정한다.

빈줄/주석

  • 그룹을 감싸고 있는 영역을 구분하기 위해 그룹간 1줄씩 빈 줄을 만드는 것을 허용한다.
  • 빈 줄의 경우 1줄을 초과하지 않도록 하며 개발과 연동되는 경우 주석을 사용한다.
  • 주석 기호와 주석내용 사이에는 반드시 공백 한 칸이 있어야 한다.
  • 시작과 종료 주석의 내용은 동일해야 한다.
  • 레이아웃 엘리먼트를 제외한 독립된 컨텐츠 영역의 주석 표기는 선택사항이다.

섹션 주석

  • 주석의 경우 섹션별로 그룹화 하여 작성한다.
  • 주석은 줄 바꿈으로 구분하다.

타입 선택자

  • 꼭 필요한 경우가 아니라면 태그와 함께 id, class를 지정하여 사용하지 않는다.
  • 불필요한 부모, 조상 선택자의 사용을 피하는것이 성능 및 커스터마이징에 용이하다.
/* bad */ div#section {...} div.section {...} /* good */ #section {...} .section {...}

공통 속성의 변수 선언 및 사용

  • :root 변수 선언
  • 공통으로 사용될 속성을 미리 변수에 선언한다.
  • var로 변수를 선언하고 소괄호 안에 속성 값을 입력한다.
:root { --color-black:#000; --color-gray1:##f4f4f4; --color-gray2:#eee; --color-gray3:#ddd; } .example {color:var(--color-gray1);}

속성 줄여쓰지, 속기

  • 가능하면 속성을 줄여쓴다.
/* bad */ padding:1rem 2rem 1rem 2rem; background-image:url(example.png); background-repeat:no-repeat; background-position:0 0; background-size:1rem 1rem; /* good */ padding:1rem 2rem; background:url(example.png) no-repeat 0 0/1rem;

단위

  • 기본적으로 rem 단위를 사용하여, :root 가상 클래스를 사용하여 기본 단위를 지정하여 사용한다.
  • 모바일과 같이 유동적인 레이아웃 구현시에는 % 단위도 같이 사용한다.
  • '0'값은 단위를 붙이지 않는다. 단,flex-basis는 제외
:root{ font-size:10px; } /* font-size:16px; */ font-size:1.6rem; flex:1 1 0px; margin:0; padding:0;

0 생략

  • -1과 1사이의 값에서 0을 생략할 수 있다.
border-radius:.6rem;

16진법 표기법

  • 줄임이 가능하다면 3자리로 표기하는것이 좋다.
/* bad */ background-color:#ffffff; color:#000000; /* good */ background-color:#fff; color:#000;

세미콜론(;)

  • 마지막에 선언된 속성 세미콜론을 제외해도 무방하지만 일관성 및 확장성을 위해 모든 속성 선언 끝에는 세미콜론을 붙여준다.

속성과 속성 사이의 여백

  • 속성과 속성 사이에 한칸의 여백을 두어 가독성을 높여준다.

따옴표

  • 작은 따옴표의 경우 폰트, filter, content외에는 사용하지 않는다.
  • background속성의 url값에는 따옴표를 사용하지 않는다.

강제 적용

  • !important는 과도하게 사용하지 않는다.
